# 빅카인즈 실측 파이프라인 현황 (감시 보도 비중 LLM 분류)

> 목적: 보고서 주장 "감시가 중요(4.43)라면서 실제 지면엔 소수뿐"을 빅카인즈 2025 기사로 **실측** 뒷받침.
> 방법: 기사를 LLM(Gemini)으로 3분류(감시 / 정보·의제 / 해결책) → 감시 비중 산출.
> 신뢰도: 사람 2명 이중 라벨링으로 만든 gold 150건에 LLM 정확도를 채점해 방법론 방어.
> 코드: `bigkinds/01~04`. 실행 폴더는 `bigkinds/` 안. (git 미추적 — 원자료 용량)

## 지금 막힌 곳 = ④ 이중 라벨링 (사람 손 필요)
- **방식 확정: 이중 라벨링** — A·B 두 사람이 **각자 150건 전체** 독립 라벨링 (75/75 분할 아님).
  근거: κ 신뢰도 확보 → 보고서에 "코더 2명 + AI 삼자 비교"로 방어. (2026-07-13 결정)
- 라벨링 가이드: **`notes/15-labeling-guide.md`** (두 사람에게 전달할 것. LLM 프롬프트와 동일 기준).
- 라벨 값은 정확히 `감시` / `정보·의제`(가운뎃점) / `해결책` 셋 중 하나.
- ⚠️ 서로 상의 말고 각자 먼저 끝낼 것 (미리 맞추면 일치도 무의미).

## 라벨링 끝난 뒤 실행 순서
1. `python 04_score.py`
   → 일치율·Cohen's κ 출력. 불일치 건은 `out/reconcile_needed.csv` 로 나옴.
2. 불일치 건 두 사람이 상의 → `out/evalset_150.csv` 의 `최종라벨` 칸 채움.
3. `python 02_classify.py out/evalset_150.csv` → `classified_evalset_150.csv` (평가셋 LLM 분류).
   (GEMINI_API_KEY 필요. 무료 등급 일일 한도 걸리면 다음날 같은 명령 재실행 = 이어서 함.)
4. `python 04_score.py` 재실행 → gold vs LLM 정확도·클래스별 F1·혼동행렬·**'감시' 정밀도/재현율**·출처별 정확도.
5. 정확도 납득되면 본분류: `python 02_classify.py out/sample_2448.csv` → 2,448건 감시 비중 산출.

## 의견 장르(사설·칼럼) 제외 — **확정 규칙** (2026-07-13)
- 3분류 rubric은 '보도기사의 역할'용. 사설·칼럼은 감시·해결책으로 구조적 쏠림 → 감시 비중 부풀림 → 제외.
- 표집 단계에서 빅카인즈 '오피니언' 대분류는 이미 빠짐. 잔여(경제면 사설 등 정치·경제·사회로 태깅된 것)를
  **제목 마커로 제거**: 공용 모듈 `bigkinds/filters.py` 의 `is_opinion(title)`.
  - 실측: sample_2448 71건(2.9%), evalset_150 5건(3.3%). 오탐 0(대괄호/꺾쇠 안 칼럼명만 인정, 정밀도 우선).
- 적용 위치:
  - `04_score.py` — 이미 물림. 채점·κ·gold·출처집계 모두 보도기사만. (라벨링 파일은 안 건드림, 채점 때만 제외)
  - **본분류(2,448)** — LLM 넣기 전 `df = df[~df['제목'].map(is_opinion)]` 로 드롭할 것.
  - **⑦ 인용지표(05)** — 같은 필터 적용 후 집계.
- 세컨더리(사설 따로 분석: 말로 주장 vs 지면 실행 격차)는 **열지 않기로** 함(2026-07-13, 범위 관리).

## 판정 근거·주의
- `sample_2448` 은 층화 랜덤이라 감시가 ~10% → 랜덤 표본만으론 감시 정확도 검증 불가.
  그래서 평가셋 150은 감시·해결책 후보를 **오버샘플링**(watchdog 35+dandok 15, solution 30, random 70).
  정확도는 이 오버샘플 셋으로, **감시 비중(%)** 은 본분류 `sample_2448`(비오버샘플)로 낼 것 — 섞지 말 것.
- `출처` 칸은 라벨링 파일에서 뺐다(정답 힌트 차단). evalset_150.csv 엔 남아 있으니 채점·집계에만 사용.

## ⑦ (보류) 인용 저널리즘 지표 — **라벨링·본분류 끝난 뒤 착수** (2026-07-13 결정)
목적: "따옴표 저널리즘"(발언 받아쓰기)을 전달형 저널리즘의 **표층 지표**로 정량화 →
설문(보도자료 74%)·역할분류(감시 격차)와 **삼각검증**. 역할분류와 겹치지 않는 별도 축.

- 프로토타입 이미 확인됨 (sample_2448, 리드200자+제목, **API 불필요·정규식**):
  - 제목에 따옴표 **59.4%** / 리드 직접인용쌍 24.8% / 리드 발화동사 1개+ 32.4% / 전달형표층(제목OR리드) **63.5%**
- 만들 것: `05_quote_metrics.py` — 기사별 지표화(제목따옴표 bool, 리드 인용쌍 수, 발화동사 수,
  선택: 인용주체 권력/시민 직함사전 분류) → **인용밀도 × 역할(감시/정보·의제/해결책) 교차표**.
  가설: '정보·의제 × 고인용' 칸이 다수 = 받아쓰기형의 정량 실체.
- 프레이밍 원칙: **인용 밀도 ≠ 나쁨.** 반드시 역할과 교차해 해석(감시는 책임추궁형 인용).
  검증여부 판정은 LLM 역할분류가 담당, 인용지표는 표층 증상 — 역할 분담 명시.
- ⚠️ 붙여진 설명 속 특정 출처("기자협회 내란사태 보도" 등)는 리포트 삽입 전 실제 확인 필수(개념은 확립됨).
